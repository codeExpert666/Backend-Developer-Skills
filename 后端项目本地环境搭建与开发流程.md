# 后端项目本地环境搭建与开发流程

适用于公司 Windows 电脑。除明确标注为“Windows PowerShell”的命令外，其余命令都在公司定制的 Ubuntu 24.04 WSL 中执行。

命令中的 `{{...}}` 是公司参数，正式使用前必须替换。

## 一、填写公司参数

| 参数 | 含义 |
| --- | --- |
| `{{WSL_DISTRO_NAME}}` | 公司 WSL 发行版名称 |
| `{{WINDOWS_USER_NAME}}` | Windows 用户目录名称 |
| `{{REPO_URL}}` | 代码仓地址 |
| `{{PROJECT_DIR}}` | 项目目录名 |
| `{{GIT_USER_NAME}}` | Git 提交姓名 |
| `{{GIT_USER_EMAIL}}` | Git 公司邮箱 |
| `{{FEATURE_BRANCH_PATTERN}}` | 功能分支名，例如 `feature/123-login` |
| `{{MYSQL_SERVICE}}` | `compose.yaml` 中的 MySQL 服务名 |
| `{{REDIS_SERVICE}}` | `compose.yaml` 中的 Redis 服务名 |
| `{{MYSQL_DATABASE}}` | 业务数据库名 |
| `{{MYSQL_MAJOR_VERSION}}` | MySQL 主版本 |
| `{{MYSQL_PASSWORD_ENV}}` | MySQL 容器内保存 root 密码的环境变量名 |
| `{{TEST_DB_HOST}}` | 测试数据库地址，仅导出人员使用 |
| `{{TEST_DB_PORT}}` | 测试数据库端口，仅导出人员使用 |
| `{{TEST_DB_EXPORT_USER}}` | 测试数据库导出账号，仅导出人员使用 |
| `{{DB_SQL_FILE_NAME}}` | 小组内传递的 `.sql` 文件名 |
| `{{KEY_DATA_VALIDATION_SQL}}` | 导入后的关键数据验证 SQL |
| `{{BACKEND_MAIN_CLASS}}` | 后端启动类 |
| `{{BACKEND_PROFILE}}` | 后端本地配置或 Profile |
| `{{BACKEND_VERIFY_COMMAND}}` | 后端验证命令，例如 `mvn test` |
| `{{BACKEND_HEALTH_URL}}` | 后端健康检查地址 |
| `{{FRONTEND_INSTALL_COMMAND}}` | 前端依赖安装命令 |
| `{{FRONTEND_START_COMMAND}}` | 前端启动命令 |
| `{{FRONTEND_URL}}` | 前端访问地址 |
| `{{DOCKER_DAEMON_PROXY_URL}}` | Docker daemon 代理地址，按需填写 |
| `{{DOCKER_NO_PROXY}}` | Docker 不走代理的地址，按需填写 |

开始前准备：公司 WSL 安装脚本、Maven `settings.xml`、代码仓权限、项目 `.env.local`、测试数据库 SQL、IDEA、VS Code 和 VS Code WSL 扩展。

## 二、安装 WSL 并配置代理

### 2.1 安装公司 WSL

1. 运行公司提供的图形化安装脚本。
2. 选择 Ubuntu 24.04 和公司要求的定制选项。
3. 如果脚本要求重启，点击同意。
4. 重启后脚本会自动打开，再次点击安装。

> **截图占位：公司 WSL 安装脚本界面**
>
> 正式发布时插入截图，并标出 Ubuntu 24.04、必选项和安装按钮。

在 Windows PowerShell 中验证：

~~~powershell
wsl -l -v
wsl -d "{{WSL_DISTRO_NAME}}"
~~~

在 WSL 中验证：

~~~bash
cat /etc/os-release
ps -p 1 -o comm=
~~~

应显示 Ubuntu 24.04，且 PID 1 为 `systemd`。

### 2.2 使用 Windows 代理

适用于 Windows 11 22H2 及以上版本。在 Windows PowerShell 中执行：

~~~powershell
notepad "$env:USERPROFILE\.wslconfig"
~~~

合并以下配置，不要覆盖文件中的其他设置：

~~~ini
[wsl2]
autoProxy=true
~~~

保存后执行：

~~~powershell
wsl --shutdown
wsl -d "{{WSL_DISTRO_NAME}}"
~~~

在 WSL 中验证：

~~~bash
env | cut -d= -f1 | grep -iE '^(http|https|no)_proxy$'
curl -I https://ubuntu.com/
~~~

如果普通 `curl` 正常，但 `sudo apt update` 超时，执行：

~~~bash
sudo visudo -f /etc/sudoers.d/wsl-proxy-env
~~~

添加一行：

~~~sudoers
Defaults env_keep += "http_proxy https_proxy no_proxy HTTP_PROXY HTTPS_PROXY NO_PROXY"
~~~

保存后执行：

~~~bash
sudo chmod 0440 /etc/sudoers.d/wsl-proxy-env
sudo visudo -c
sudo apt update
~~~

不要使用会保留整个用户环境的 `sudo -E`。

## 三、安装 Java、Maven 和 Node.js

### 3.1 Java 17

~~~bash
sudo apt update
sudo apt install -y openjdk-17-jdk

java_home="$(dirname "$(dirname "$(readlink -f "$(command -v javac)")")")"
printf '\nexport JAVA_HOME="%s"\nexport PATH="$JAVA_HOME/bin:$PATH"\n' "$java_home" >> "$HOME/.profile"
. "$HOME/.profile"

java -version
javac -version
~~~

两条版本命令都应显示 17。

### 3.2 Maven 3.9.0

~~~bash
MAVEN_VERSION="3.9.0"
cd /tmp
curl -fLO "https://archive.apache.org/dist/maven/maven-3/$MAVEN_VERSION/binaries/apache-maven-$MAVEN_VERSION-bin.tar.gz"

sudo tar -xzf "apache-maven-$MAVEN_VERSION-bin.tar.gz" -C /opt
sudo ln -sfn "/opt/apache-maven-$MAVEN_VERSION" /opt/maven

printf '\nexport MAVEN_HOME="/opt/maven"\nexport PATH="$MAVEN_HOME/bin:$PATH"\n' >> "$HOME/.profile"
. "$HOME/.profile"
~~~

将公司 `settings.xml` 放到 Windows 下载目录，然后执行：

~~~bash
install -d -m 700 "$HOME/.m2"
install -m 600 \
  "/mnt/c/Users/{{WINDOWS_USER_NAME}}/Downloads/settings.xml" \
  "$HOME/.m2/settings.xml"

mvn -version
~~~

输出必须同时包含 Maven 3.9.0 和 Java 17。不要把 `settings.xml` 提交到 Git。

### 3.3 Node.js 24.13.0

定制 WSL 已安装 nvm：

~~~bash
export NVM_DIR="${NVM_DIR:-$HOME/.nvm}"
. "$NVM_DIR/nvm.sh"

nvm install 24.13.0
nvm alias default 24.13.0
nvm use default

node --version
npm --version
~~~

`node --version` 应输出 `v24.13.0`。

## 四、安装 Docker Engine

公司不允许安装 Docker Desktop。Docker 必须安装在 WSL 中。

~~~bash
sudo apt remove -y $(dpkg --get-selections \
  docker.io docker-compose docker-compose-v2 docker-doc \
  podman-docker containerd runc | cut -f1)

sudo apt update
sudo apt install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

sudo tee /etc/apt/sources.list.d/docker.sources >/dev/null <<EOF
Types: deb
URIs: https://download.docker.com/linux/ubuntu
Suites: $(. /etc/os-release && echo "${UBUNTU_CODENAME:-$VERSION_CODENAME}")
Components: stable
Architectures: $(dpkg --print-architecture)
Signed-By: /etc/apt/keyrings/docker.asc
EOF

sudo apt update
sudo apt install -y \
  docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

sudo systemctl enable --now docker
sudo usermod -aG docker "$USER"
~~~

退出并重新进入 WSL，使用户组生效：

~~~bash
exit
~~~

在 Windows PowerShell 中执行：

~~~powershell
wsl -d "{{WSL_DISTRO_NAME}}"
~~~

回到 WSL 后验证：

~~~bash
docker run --rm hello-world
docker version
docker compose version
~~~

### Docker 拉取镜像仍然超时

仅在 `curl` 正常但 `docker pull` 超时时配置 daemon 代理：

~~~bash
sudo systemctl edit docker.service
~~~

添加：

~~~ini
[Service]
Environment="HTTP_PROXY={{DOCKER_DAEMON_PROXY_URL}}"
Environment="HTTPS_PROXY={{DOCKER_DAEMON_PROXY_URL}}"
Environment="NO_PROXY=localhost,127.0.0.1,::1,{{DOCKER_NO_PROXY}}"
~~~

保存后执行：

~~~bash
sudo systemctl daemon-reload
sudo systemctl restart docker
docker pull hello-world
~~~

## 五、克隆代码并启动 MySQL、Redis

~~~bash
sudo apt install -y git
git config --global user.name "{{GIT_USER_NAME}}"
git config --global user.email "{{GIT_USER_EMAIL}}"

mkdir -p "$HOME/workspace"
cd "$HOME/workspace"
git clone "{{REPO_URL}}" "{{PROJECT_DIR}}"
cd "{{PROJECT_DIR}}"

git switch develop
git pull --ff-only origin develop

test -d java_service
test -d vue_service
test -f compose.yaml
test -f .env.local
~~~

如果 `.env.local` 不在仓库中，将公司提供的文件放到项目根目录。该文件不得提交到 Git。

启动服务：

~~~bash
docker compose --env-file .env.local config --quiet
docker compose --env-file .env.local up -d \
  "{{MYSQL_SERVICE}}" \
  "{{REDIS_SERVICE}}"
docker compose --env-file .env.local ps
~~~

验证 MySQL：

~~~bash
docker compose --env-file .env.local exec -T "{{MYSQL_SERVICE}}" \
  sh -eu -c '
    MYSQL_PWD="$(printenv "{{MYSQL_PASSWORD_ENV}}")"
    export MYSQL_PWD
    exec mysqladmin --user=root ping
  '
~~~

如果尚未就绪，等待几秒后重试。

## 六、导入测试数据库 SQL

### 6.1 导出 SQL，仅导出人员执行

在 Windows PowerShell 中执行：

~~~powershell
mysqldump.exe --version
$SqlFile = Join-Path $env:USERPROFILE "Downloads\{{DB_SQL_FILE_NAME}}"

mysqldump.exe --host="{{TEST_DB_HOST}}" --port={{TEST_DB_PORT}} --user="{{TEST_DB_EXPORT_USER}}" --password --single-transaction --routines --triggers --events --set-gtid-purged=OFF --no-tablespaces --databases "{{MYSQL_DATABASE}}" --result-file="$SqlFile"

if ($LASTEXITCODE -ne 0) {
  throw "mysqldump failed with exit code $LASTEXITCODE"
}
~~~

`mysqldump.exe` 的版本应与 `{{MYSQL_MAJOR_VERSION}}` 兼容。

将生成的 `{{DB_SQL_FILE_NAME}}` 直接在小组内传递，不压缩，也不提交到 Git。

### 6.2 直接导入 MySQL 容器

将 SQL 文件保存到 Windows 下载目录，然后在 WSL 中执行：

~~~bash
cd "$HOME/workspace/{{PROJECT_DIR}}"

sql_file="/mnt/c/Users/{{WINDOWS_USER_NAME}}/Downloads/{{DB_SQL_FILE_NAME}}"
test -s "$sql_file"

docker compose --env-file .env.local exec -T "{{MYSQL_SERVICE}}" \
  sh -eu -c '
    MYSQL_PWD="$(printenv "{{MYSQL_PASSWORD_ENV}}")"
    export MYSQL_PWD
    exec mysql --user=root
  ' < "$sql_file"
~~~

验证导入：

~~~bash
docker compose --env-file .env.local exec -T "{{MYSQL_SERVICE}}" \
  sh -eu -c '
    MYSQL_PWD="$(printenv "{{MYSQL_PASSWORD_ENV}}")"
    export MYSQL_PWD
    exec mysql --user=root --batch --table --execute="
      SELECT VERSION() AS mysql_version;
      SELECT COUNT(*) AS table_count
      FROM information_schema.tables
      WHERE table_schema = '\''{{MYSQL_DATABASE}}'\'';
      {{KEY_DATA_VALIDATION_SQL}}
    "
  '
~~~

> `docker compose down -v` 会删除 MySQL、Redis 等本地卷。只有确定本地数据都可以删除时才能执行。

## 七、启动后端和前端

### 7.1 IDEA 启动后端

在 WSL 中执行：

~~~bash
cd "$HOME/workspace/{{PROJECT_DIR}}/java_service"
wslpath -w "$PWD"

java -version
mvn -version
{{BACKEND_VERIFY_COMMAND}}
~~~

复制 `wslpath` 输出，在 Windows IDEA 中打开项目，然后配置：

- Project SDK：WSL 中的 Java 17。
- Maven home：`/opt/maven`。
- User settings：`~/.m2/settings.xml`。
- 启动类：`{{BACKEND_MAIN_CLASS}}`。
- 本地配置或 Profile：`{{BACKEND_PROFILE}}`。

启动后访问 `{{BACKEND_HEALTH_URL}}`。

### 7.2 VS Code 启动前端

Windows VS Code 安装 WSL 扩展，然后在 WSL 中执行：

~~~bash
cd "$HOME/workspace/{{PROJECT_DIR}}/vue_service"
code .
~~~

在 VS Code 的 WSL 终端中执行：

~~~bash
node --version
{{FRONTEND_INSTALL_COMMAND}}
{{FRONTEND_START_COMMAND}}
~~~

访问 `{{FRONTEND_URL}}`。

## 八、开发和提交 MR

开始新任务：

~~~bash
cd "$HOME/workspace/{{PROJECT_DIR}}"
git switch develop
git pull --ff-only origin develop
git switch -c "{{FEATURE_BRANCH_PATTERN}}"
~~~

开发完成后：

~~~bash
git status
git diff

cd "$HOME/workspace/{{PROJECT_DIR}}/java_service"
{{BACKEND_VERIFY_COMMAND}}

cd "$HOME/workspace/{{PROJECT_DIR}}"
~~~

使用 `git add` 只暂存本任务文件，然后提交并推送：

~~~bash
git commit -m "feat: 完成功能说明"
git push -u origin "$(git branch --show-current)"
~~~

创建 MR，目标分支选择 `develop`。不要直接在 `develop` 上开发或提交。

## 九、日常启停

启动依赖服务：

~~~bash
cd "$HOME/workspace/{{PROJECT_DIR}}"
docker compose --env-file .env.local up -d \
  "{{MYSQL_SERVICE}}" \
  "{{REDIS_SERVICE}}"
~~~

停止依赖服务：

~~~bash
docker compose --env-file .env.local stop
~~~

查看日志：

~~~bash
docker compose --env-file .env.local logs --tail 100
~~~

## 十、常见问题

| 问题 | 处理 |
| --- | --- |
| WSL 没有代理 | 检查 `.wslconfig`，执行 `wsl --shutdown` 后重新进入 |
| `sudo apt update` 超时 | 检查 `/etc/sudoers.d/wsl-proxy-env` 和 `sudo visudo -c` |
| Maven 无法下载依赖 | 检查 `~/.m2/settings.xml` 和 `mvn -version` |
| `docker pull` 超时 | 配置 Docker daemon 代理并重启 Docker |
| MySQL 未就绪 | 执行 `docker compose --env-file .env.local logs --tail 200 "{{MYSQL_SERVICE}}"` |
| IDEA 或 VS Code 版本不对 | 确认终端运行在 WSL，并重新检查 Java、Maven、Node.js 版本 |
