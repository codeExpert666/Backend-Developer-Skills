本文是 Ghostty + Zsh + Antidote + Starship + Atuin + zoxide + fzf 这套终端环境的总入口。它不依赖完整的 Zsh 框架，而是让每个工具只承担一项明确职责，以便在 macOS、Ubuntu 桌面和远程 Ubuntu Server 之间复用配置、独立升级并逐项回退。

从零搭建请按平台进入 [[macOS 从零搭建现代终端环境]] 或 [[Ubuntu 从零搭建现代终端环境]]。已有 Oh My Zsh 的用户不要直接覆盖配置，应先阅读 [[从 Oh My Zsh 迁移到 Antidote]]。共享 Shell 骨架与维护流程分别见 [[Zsh 与 Antidote 跨机器配置管理]]、[[现代终端环境更新、验证与回退]]；专项配置再进入 [[Ghostty 常用配置与 Shell 集成]]、[[Starship 提示符配置]]、[[Atuin 命令历史管理]] 和 [[zoxide 与 fzf 导航和模糊查找]]。

## 先理解这套工具的分层

“终端体验”不是由一个框架包办，而是下列层次共同组成：

| 层次 | 工具 | 职责 | 安装与更新方式 |
| --- | --- | --- | --- |
| 图形终端 | Ghostty | 窗口、标签页、分屏、字体、颜色和终端协议 | 在本地桌面系统安装；不随 Shell 配置进入远程服务器 |
| Shell | Zsh | 解释命令、管理变量、补全和交互编辑 | 使用操作系统自带版本或系统包管理器 |
| Zsh 插件管理 | Antidote | 下载 Zsh 插件并生成快速的静态加载文件 | 从官方 Git 仓库安装，插件由清单声明 |
| 提示符 | Starship | 显示目录、Git 状态、语言环境和命令结果 | 独立二进制，由系统包管理器或官方安装器管理 |
| 历史记录 | Atuin | 保存带目录、主机、时间和退出码的历史并提供搜索界面 | 独立二进制；同步功能可选 |
| 目录导航 | zoxide | 根据使用频率快速跳转到访问过的目录 | 独立二进制 |
| 模糊选择 | fzf | 为文件、目录和其他列表提供模糊查找 | 独立二进制 |

> [!important] Antidote 不是通用软件包管理器
> Antidote 只管理 `zsh-autosuggestions`、`zsh-syntax-highlighting` 等 Zsh 插件。Starship、Atuin、zoxide、fzf 和 Ghostty 都是独立程序，必须由 Homebrew、apt、官方安装器或相应平台的软件包系统安装与更新。

## 为什么这里选择 Antidote，而不是 Zimfw

两者都能用声明文件在多台机器重建插件，并生成快速的静态初始化脚本；区别主要在配置的所有权。Antidote 定位为插件管理器，`.zshrc` 仍由使用者完整编排。Zimfw 同时是模块化 Zsh 框架与模块管理器，由 `.zimrc` 声明模块，并生成自己的 `init.zsh`。

| 需求 | 更自然的选择 | 原因 |
| --- | --- | --- |
| Ghostty、Starship、Atuin、zoxide、fzf 已各自独立，希望自己控制完整加载顺序 | Antidote | 只补齐 Zsh 插件下载与静态加载，不再引入第二套行为模块和主题体系 |
| 希望框架同时提供补全、输入、环境、工具函数和主题等成套模块 | Zimfw | 官方模块与 `.zimrc` 能减少手工组织 `.zshrc` 的工作 |

因此这里选择 Antidote 不是断言它在所有场景都“优于” Zimfw，而是这套组件化方案已经把提示符、历史和导航拆给独立工具，Antidote 的职责边界更贴合。跨机器维护的可靠性最终仍取决于是否同步声明文件、排除本机状态，并保留可验证的恢复流程。

## Ghostty 应安装在哪里

Ghostty 是带图形界面的终端模拟器，应运行在你眼前的本地桌面设备上，而不是因为使用了 SSH 就安装到每台远程主机。

| 使用场景 | Ghostty 的位置 | 远端需要安装什么 |
| --- | --- | --- |
| Mac 直接开发 | 安装在 Mac | 不涉及远端 |
| Mac 通过 SSH 连接 Ubuntu Server | 只安装在 Mac | Zsh、Antidote 和需要的命令行工具 |
| Ubuntu Desktop 本地开发 | 可安装在 Ubuntu Desktop | 不涉及远端 |
| 无图形界面的 Ubuntu Server | 不安装 Ghostty | Zsh、Antidote、Starship、Atuin、zoxide、fzf 按需求安装 |

Ghostty 官方目前只直接发布 macOS 预编译二进制；Linux 版本主要由发行版或社区维护者构建。Ubuntu Desktop 用户应先评估包来源和维护者，再决定使用发行版包、社区包或自行构建，不能把第三方一行安装脚本当成 Ghostty 官方二进制。远程 Server 没有图形桌面时，更没有安装 Ghostty 的必要。

## 适合与不适合的使用对象

这套方案适合：

- 希望在多台 macOS、Ubuntu 或 SSH 主机上维护同一份 Zsh 配置；
- 希望提示符、历史、导航和模糊查找可以独立替换；
- 愿意理解少量 Zsh 启动文件边界，并用 Git 管理自己的配置；
- 觉得 Oh My Zsh 功能过多、启动链路不够透明，或只使用其中少量插件。

以下场景更适合继续使用 [[Oh My Zsh 安装概览]]：

- 只维护一台个人电脑，希望安装后立即获得主题和常用别名；
- 不想自行组织 `.zshrc`、补全、插件清单和各工具初始化顺序；
- 当前 Oh My Zsh 已稳定运行，且没有跨机器维护或启动性能问题。

选择 Antidote 不是因为 Oh My Zsh “不能用”，而是把完整框架改成一组职责清楚、可组合的组件。现有 Oh My Zsh 笔记仍保留为更简单的入门路线。

## 跨机器目录模型

推荐让 Zsh 配置进入 XDG 目录，并只在主目录保留一个很小的引导文件：

~~~text
~/.zshenv
~/.config/
├── zsh/
│   ├── .zprofile
│   ├── .zshrc
│   ├── .zsh_plugins.txt
│   ├── common.zsh
│   ├── macos.zsh
│   ├── linux.zsh
│   ├── local.zprofile
│   ├── local.zsh
│   └── snapshots/
│       └── antidote.txt
├── ghostty/
│   └── config.ghostty
├── atuin/
│   └── config.toml
└── starship.toml
~/.local/share/antidote/
~~~

`~/.zshenv` 只设置 `ZDOTDIR` 和非交互 SSH 命令也必须看到的最小 PATH。设置后，Zsh 会从 `~/.config/zsh` 读取 `.zprofile` 与 `.zshrc`：

- `.zprofile`：登录 Shell 所需的环境初始化，例如 Homebrew `shellenv`；
- `.zshrc`：交互选项、补全、插件、按键、Atuin、zoxide 和提示符；
- `common.zsh`：跨平台别名和函数；
- `macos.zsh`、`linux.zsh`：平台专属配置；
- `local.zprofile`：机器私有的登录环境，不提交到配置仓库；
- `local.zsh`：代理、公司 SDK 或机器私有设置，不提交到配置仓库；
- `.zsh_plugins.txt`：Antidote 的声明式插件清单。

> [!warning] `ZDOTDIR` 的引导位置不能搬错
> Zsh 在知道 `ZDOTDIR` 之前仍从主目录寻找 `~/.zshenv`。因此应保留这个最小引导文件；不能只把 `.zshenv` 放进 `~/.config/zsh`，否则全新会话不会自动发现新的配置目录。

## 哪些内容适合跨机器同步

配置仓库应保存“声明和配置”，不保存可重新下载的缓存、使用记录或密钥：

| 内容 | 建议纳入 Git | 原因 |
| --- | --- | --- |
| `~/.zshenv`、Zsh 配置与 `.zsh_plugins.txt` | 是 | 决定启动行为与插件集合 |
| Antidote snapshot | 可选 | 固定插件提交，便于重现已验证状态 |
| `starship.toml`、Ghostty 配置 | 是 | 可复用的界面与提示符设置 |
| 已清理敏感项的 Atuin `config.toml` | 是 | 保存搜索和过滤偏好 |
| `local.zsh`、代理、令牌和公司私有路径 | 否 | 机器差异或敏感信息 |
| `~/.local/share/antidote` 与 Antidote 插件缓存 | 否 | 可由 Git 和清单重新生成 |
| `.zsh_plugins.zsh` | 否 | Antidote 生成的静态加载文件 |
| Atuin 数据库、会话和加密密钥 | 否 | 包含真实历史或身份材料 |
| zoxide 数据库、Starship 缓存和日志 | 否 | 运行时状态，可自动重建 |

## 固定的交互加载顺序

加载顺序决定按键绑定、补全和高亮是否稳定。推荐顺序如下：

1. `~/.zshenv` 设置 `ZDOTDIR` 和最小 PATH。
2. `.zprofile` 设置登录环境。
3. `.zshrc` 设置历史选项，加载跨平台与本机配置，并执行 `compinit`。
4. Antidote 加载普通插件和自动建议。
5. fzf 初始化，但让出 `Ctrl-R` 给 Atuin，并禁用 `Alt-C`；目录交互选择统一使用 zoxide 的 `zi`。
6. Atuin 初始化并接管 `Ctrl-R`。
7. zoxide 在 `compinit` 之后初始化。
8. Starship 初始化提示符。
9. 最后显式 `source` `zsh-syntax-highlighting`。

语法高亮不能只是“写在 Antidote 清单最后”。因为清单加载后仍会创建 fzf、Atuin、zoxide 和 Starship 的交互部件，`zsh-syntax-highlighting` 必须在这些步骤之后真正执行。推荐在清单中使用：

~~~text
zsh-users/zsh-autosuggestions
zsh-users/zsh-syntax-highlighting kind:clone
~~~

然后在 `.zshrc` 最后加载：

~~~zsh
if (( $+functions[antidote] )); then
  zsh_highlight_root="$(antidote path zsh-users/zsh-syntax-highlighting 2>/dev/null)"
  if [[ -r "$zsh_highlight_root/zsh-syntax-highlighting.zsh" ]]; then
    source "$zsh_highlight_root/zsh-syntax-highlighting.zsh"
  fi
  unset zsh_highlight_root
fi
~~~

## 推荐的安装与阅读顺序

1. 记录当前 Shell、PATH、Zsh 配置和已安装工具，创建可识别的备份。
2. 安装或确认 Zsh、Git 与 curl；远程主机先保留一个备用 SSH 会话。
3. 仅在本地桌面安装 Ghostty；Ubuntu Server 跳过。
4. 用平台包管理器或官方安装器安装 Starship、Atuin、zoxide 与 fzf。
5. 将 Antidote 从官方仓库克隆到 `${XDG_DATA_HOME:-$HOME/.local/share}/antidote`。
6. 建立 `~/.zshenv` 与 `~/.config/zsh`，先用最小插件清单启动。
7. 导入 Atuin 或 zoxide 的旧数据；同步和主题美化放到基础验证之后。
8. 按 [[现代终端环境更新、验证与回退]] 建立更新、快照和恢复流程。

不要一次加入十几个插件、复杂 Starship preset 和大量 Ghostty 快捷键。先让每层独立通过验证，再逐项增强，才能在故障时判断问题属于终端、Shell、插件还是独立工具。

## 最小成功标准

新开一个终端后执行：

~~~zsh
printf 'login shell: %s\n' "$SHELL"
printf 'current shell: '
ps -p $$ -o comm=
printf 'ZDOTDIR: %s\n' "$ZDOTDIR"

for command_name in zsh git starship atuin zoxide fzf; do
  if (( $+commands[$command_name] )); then
    printf '%-10s %s\n' "$command_name" "$commands[$command_name]"
  else
    printf '%-10s missing\n' "$command_name"
  fi
done

antidote home
atuin doctor
zoxide query --list | head

for startup_file in "$HOME/.zshenv" "$ZDOTDIR/.zprofile" "$ZDOTDIR/.zshrc"; do
  [[ -f "$startup_file" ]] && zsh -n "$startup_file"
done
~~~

最小成功状态是：当前进程为 Zsh，`ZDOTDIR` 指向 `~/.config/zsh`，六个命令行组件都能找到，Antidote 能输出缓存目录，Atuin 自检没有阻断性错误，Zsh 配置通过语法检查。zoxide 在尚未积累目录数据时输出为空是正常现象。

Ghostty 只在安装它的本地图形设备检查：

~~~bash
ghostty +version
~~~

如果 macOS 中命令暂时不在 PATH，可先从“应用程序”启动 Ghostty，再在 Ghostty 窗口内验证；这不影响远端 Ubuntu 的安装结果。

## 与 Oh My Zsh 路线的关系

- 新机器、愿意维护模块化配置：从本文进入现代终端路线。
- 现有 Oh My Zsh 用户：按 [[从 Oh My Zsh 迁移到 Antidote]] 并行准备，不删除原配置。
- 只想快速得到主题和常用插件：继续 [[Oh My Zsh 安装概览]]。
- 两条路线都应遵守同一原则：备份后修改，保留一个可用会话，新会话验证通过后再清理旧环境。
