本文是 Linux 命令行基础的统一阅读入口。目标不是背下一部命令手册，而是建立三种能力：能拆解一条命令、能判断应该掌握到什么程度、能在忘记参数时自己查回来。

本文以 Ubuntu Server 中常见的 Bash 环境为主。其他 Shell 的核心概念相近，但具体语法和启动文件可能不同。

## 1. 先区分五个容易混在一起的概念

一条命令能显示结果，通常有多个层次共同参与：

```text
键盘输入
  -> 终端：负责显示字符和传递输入输出
  -> Shell：解析引号、变量、管道、重定向等语法
  -> 命令或程序：完成文件、网络、服务或项目操作
  -> Linux 内核：提供进程、文件、网络等系统能力
```

| 概念 | 解决的问题 | 例子 |
| --- | --- | --- |
| 终端 | 在哪里输入并查看结果 | 本地终端窗口、SSH 会话、虚拟机控制台 |
| Shell | 怎样理解和组合输入 | Bash、Zsh、Dash |
| Shell 内建命令 | 必须或适合由 Shell 自己完成的操作 | `cd`、`export`、`type`、`read` |
| Linux/GNU 系统命令 | 操作系统和基础用户空间工具 | `ls`、`cp`、`grep`、`ps`、`ip` |
| 项目 CLI | 管理某个开发工具或项目 | `git`、`docker`、`go`、`mvn` |

因此，下面这行不应被看成一个需要整体背诵的“Linux 命令”：

```bash
journalctl -u ssh.service -b -n 80 --no-pager | tail -n 20
```

它至少包含：

- Shell 管道语法 `|`。
- `journalctl` 命令及其选项、参数。
- `tail` 命令及其选项、参数。
- 两个进程之间的标准输出和标准输入连接。

先理解这些层次，再学习具体命令，会比逐行死记更容易迁移到新场景。命令拆解方法见 [[Shell 命令结构、类型与帮助系统]]，组合规则见 [[Shell 标准流、管道、重定向与退出状态]]。

## 2. 三档掌握标准

“掌握”并不只有会与不会两种状态。本系列统一使用三档目标：

| 层级 | 达成标准 | 学习方式 |
| --- | --- | --- |
| 必须熟练 | 不查资料也能写出基础骨架，能预判常见风险 | 反复在安全目录练习，进行间隔回忆 |
| 理解会查 | 知道它解决什么问题、常用骨架是什么，参数忘了会查 | 按真实任务使用，保留少量典型模式 |
| 认识即可 | 看到时能判断大致目的和风险 | 用到时查官方帮助，不提前背参数 |

这个分级取决于当前学习阶段，不是命令的永久等级。例如刚开始只需认识复杂 `awk`，当日志处理成为日常工作后，它可以升级为“理解会查”。

### 必须熟练的核心

第一阶段优先掌握以下能力，不要求一次记住所有选项：

- 定位和导航：`pwd`、`cd`、`ls`，以及绝对路径、相对路径、`.`、`..`、`~`。
- 基础文件操作：`mkdir`、`touch`、`cp`、`mv`、`rm`。
- 查看文本：`cat`、`less`、`head`、`tail`。
- 搜索：熟练 `grep` 的基础骨架；能看懂 `find` 的“起始路径 → 条件 → 动作”结构，具体条件理解会查。
- 身份与权限判断：`whoami`、`id`、`sudo`、`chmod` 的基本用途。
- Shell 阅读：变量、引号、通配符、命令替换、管道、重定向、退出状态。
- 自助查询：`type`、`--help`、`help`、`man`。

### 理解会查的常用命令族

以下按**整个命令族**分级：需要知道它们解决什么问题，但不必脱离场景背完参数。对应专题仍会把其中少量最小、高频骨架标为“必须熟练”；这不代表需要背诵整个命令族。

- 软件包：`apt`、`dpkg-query`。
- 进程与资源：`ps`、`pgrep`、`top`、`kill`、`free`、`df`、`du`。
- 服务与日志：`systemctl`、`journalctl`。
- 网络与远程访问：`ip`、`ss`、`ssh`、`scp`、`rsync`。
- 权限和路径诊断：`stat`、`namei`、`realpath`。

### 认识即可的内容

- 复杂 `sed`、`awk`、正则表达式和 `find -exec` 组合。
- 不常用的输出格式化选项。
- 只为某次故障出现的底层诊断命令。
- 项目 CLI 中低频的子命令和开关。

专家也会查手册。真正重要的是知道应该找哪个命令族、怎样确认参数含义，以及执行前如何判断风险。

## 4. 按问题查命令，而不是按字母背命令

| 我想解决的问题 | 先想到的工具 | 当前目标 |
| --- | --- | --- |
| 我在哪里、目录里有什么 | `pwd`、`ls` | 必须熟练 |
| 进入另一个目录 | `cd` | 必须熟练 |
| 创建、复制、移动、删除 | `mkdir`、`cp`、`mv`、`rm` | 必须熟练 |
| 快速查看文件 | `cat`、`less`、`head`、`tail` | 必须熟练 |
| 在文本中找内容 | `grep` | 必须熟练 |
| 按名称、类型或时间找文件 | `find` | 理解会查 |
| 查看身份和权限 | `whoami`、`id`、`ls -l`、`stat` | 必须熟练基础骨架 |
| 查看进程和资源 | `ps`、`top`、`free`、`df`、`du` | 理解会查 |
| 查看服务和日志 | `systemctl`、`journalctl` | 理解会查 |
| 查看地址、路由和端口 | `ip`、`ss` | 理解会查 |
| 连接远程主机 | `ssh` | 理解会查 |
| 安装 Ubuntu 软件包 | `apt` | 理解会查 |
| 不知道命令来自哪里 | `type`、`command -V` | 必须熟练 |
| 不知道参数怎么写 | `help`、`--help`、`man` | 必须熟练 |

同一个命令可以出现在多个问题中。例如 `tail` 既能查看文件末尾，也常用 `-f` 持续查看日志。学习时保留“问题 → 命令骨架”的关系，比孤立记忆参数更有用。

## 5. 先判断风险，再按 Enter

| 风险等级 | 特征 | 处理方式 |
| --- | --- | --- |
| 只读 | 查询状态，不改变文件或系统 | 仍需确认是否会暴露敏感信息 |
| 当前会话变化 | 改目录、变量、Shell 选项 | 退出会话通常恢复，但可能影响后续命令 |
| 用户目录变化 | 创建、覆盖或删除当前用户文件 | 先打印目标路径，在临时目录练习 |
| 系统变化 | 使用 `sudo`、安装软件、改服务或网络 | 先读完整命令、准备验证和恢复路径 |
| 数据破坏 | 递归删除、覆盖磁盘、清空数据库 | 不从聊天或网页直接复制执行，逐项核对目标 |

特别警惕以下信号：

- `sudo`：后续命令可能以更高权限运行，但它的作用范围需要按 Shell 语法判断。
- `rm`、`mv`、`cp`：可能删除或覆盖已有文件。
- `>`：目标存在时会先截断文件。
- `chmod -R`、`chown -R`：递归修改范围可能远大于预期。
- `curl ... | sh`：下载内容未经过本地审阅就交给 Shell 执行。
- 未引用变量：变量为空、包含空格或通配符时，目标可能变化。

## 6. 每次遇到新命令的学习闭环

### 第一步：拆解

依次标出：

1. Shell 语法。
2. 命令名。
3. 选项。
4. 选项对应的参数。
5. 普通参数或路径。
6. 输入来自哪里、输出去哪里。

### 第二步：确认用途和风险

先回答三个问题：

- 它读取什么？
- 它修改什么？
- 如果变量为空或参数写错，最坏会怎样？

### 第三步：只学一个骨架

例如不要背下所有 `journalctl` 参数，只先记：

```text
journalctl [筛选对象] [时间范围] [输出方式]
```

然后把当前遇到的 `-u`、`-b`、`-n` 放回这个骨架。下一次换服务或条数时，结构仍然成立。

### 第四步：在安全范围内改一个变量

先预测结果，再改变一个选项或参数。不要一次改三处，否则很难知道是哪一处导致结果变化。

### 第五步：间隔回忆

在当天、次日、三天后和一周后，不看笔记回答：

- 这个命令解决什么问题？
- 最小骨架是什么？
- 哪些操作有风险？
- 忘记参数时从哪里查？

同一命令实际遇到三次后，再决定是否增加自己的命令卡片，避免为低频命令维护大量孤立笔记。

## 7. 第一个安全练习目录

下面的练习只在 `/tmp` 下创建一个系统分配的唯一目录，不使用 `sudo`，不会修改系统配置。目录会保留到系统清理 `/tmp`，方便你检查结果。

**执行位置：Ubuntu 主机（任意目录；会在 `/tmp` 创建练习目录）**

```bash
if practice_dir="$(mktemp -d /tmp/linux-command-study.XXXXXX)"; then
  if (
    cd -- "$practice_dir" || exit 1
    mkdir notes
    printf 'first command note\n' > notes/day-1.txt
    cp notes/day-1.txt notes/day-1-copy.txt
    ls -la notes
    grep -n 'command' notes/day-1.txt
    pwd
  ); then
    printf 'practice_dir=%s\n' "$practice_dir"
  else
    printf '%s\n' '练习已停止：未在临时目录中完成操作。' >&2
  fi
else
  printf '%s\n' '练习已停止：无法创建临时目录。' >&2
fi
```

预期结果：

- 成功后最后一行输出真实练习目录，例如 `/tmp/linux-command-study.A1b2C3`。
- `notes` 中出现两个普通文件。
- `grep` 显示包含 `command` 的第 1 行。
- `pwd` 最终显示练习目录，而不是原来的目录。

练习包含目录和文件创建，属于“限定在临时目录内的变更操作”。暂时不要急着删除；先用 `ls -la "$practice_dir"/notes` 重新检查，并尝试解释每一部分。

## 8. 向 Codex 提问时保留学习目标

```text
请按 Ubuntu Server 的 Bash 环境解释下面这条命令：

1. 区分 Shell 语法、命令、选项和参数；
2. 说明输入、标准输出和标准错误分别去哪里；
3. 说明它是只读、当前会话变化、用户文件变化还是系统变化；
4. 总结最值得掌握的一个命令骨架，不展开完整参数大全；
5. 判断属于“必须熟练、理解会查、认识即可”中的哪一档；
6. 给一个不使用 sudo、只在 mktemp 目录中操作的变体练习；
7. 告诉我怎样用当前机器上的 help、--help 或 man 重新查到答案。

命令：……
```

Codex 的解释用于建立线索；当前系统的帮助、手册和命令实际输出才是确认版本与行为的依据。
